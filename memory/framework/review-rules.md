# 点评沉淀 — 输出方法论规则

对 `fa vet` / `fa report` 输出的**长期点评**沉淀于此；`## 规则` 之后的内容会在每次
合成时注入提示词（空或以「（暂无」开头则跳过注入）。

收录原则（策展是人的事）：
- 只收长期偏好（"每次都该这样做"）。一次性纠错不进这里——内容错了改 CoT/note，
  路由错了改框架 frontmatter 的 applies/avoid，召回错了改 sectors.yaml
- 一条一行：做什么 + 为什么/反例
- 规则互相打架或超过 ~30 条时合并修剪

## 规则

- 引用 CoT 写成「C编号·触发词关键词（信号X/10）」，例：C259·学术孵化获标杆客户（信号8/10）。传导/证伪/历史/时效四个子分**不进正文**——那是链的质量分不是观点强度分，读者会误读（点评来源：2026-06-10，"证伪8"被误读为风险概率）
- 逐条校验时给每条命中标类型：**差异化优势 / 行业共性 / 风险机制** 三选一。行业标配动作的命中不算加分项（只说明公司没掉队），只有**缺失**标配才是信号；差异化优势才能支撑可信度（点评来源：2026-06-10，开源策略被当成加分项）
